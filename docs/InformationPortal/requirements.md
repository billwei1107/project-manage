# 資訊門戶 (Information Portal) 需求文檔

## 核心功能概述

資訊門戶旨在為企業內部員工提供一個集中化、結構化的客戶資訊與檔案共用管理中心。系統將依據「不同客戶」建立分類，並支援以下核心能力：

1. **客戶分類與基本資訊管理**
   - 門戶首頁或側邊欄需依據「客戶 (Client/Company)」進行分類。
   - 每個客戶擁有專屬的頁面/資料夾，內部包含：
     - **客戶基本聯絡資訊** (聯絡人、電話、地址、備註)。
     - **該客戶的專屬檔案總管**。

2. **多層次檔案總管 (File Explorer)**
   - 支援無限層級的資料夾結構 (Folders)。
   - 支援檔案上傳與管理 (上傳、下載、重新命名、刪除)。
   - **線上預覽**：支援圖片 (PNG, JPG) 直接預覽。
   - 預設可包含如「合約相關」、「交付檔案」、「需求文檔」等預設結構。

3. **外部共享連結 (Share Links)**
   - 能夠針對特定的「資料夾」產生外部共享連結。
   - 將連結提供給客戶後，客戶可直接透過該連結，將應提供的資料/檔案「上傳」至該資料夾中 (類似 Google Drive / Dropbox 的檔案收集請求)。
   - 增強考量：可設定連結有效期限、密碼保護 (可選)。

4. **系統公告與說明文件 (Announcements & Docs)**
   - 全域的佈告欄功能。
   - 支援發佈管理平台公告、系統更新日誌、操作說明文件等。
   - 支援 Rich Text (Markdown / 格式化文字) 帶有標題與內文的發佈形式。

## 前端使用體驗
- 需有類似 Finder 或 Windows 檔案總管的直覺介面。
- 支援拖曳上傳與資料夾切換。
- 麵包屑導覽 (Breadcrumbs) 顯示當前資料夾層級。

## 2026-03-02 追加開發紀錄 (GitHub 設定簡化與修復)

### 問題追蹤
- **問題描述 (Issue)**: 專案建立時回報 `Failed to save`，後端日誌顯示 `GitHub Enterprise server (https://api.github.com) with private mode enabled`。這是因為測試伺服器缺少 `GITHUB_ORG_TOKEN` 環境變數，導致 `org.kohsuke.github.GitHubBuilder` 嘗試使用預設的空值或佔位符連線失敗。
- **解決方案 (Solution)**: 
  1. 後端 `GitHubService.java` 新增判斷，若無合法 Token 則拋出清晰的 `IllegalArgumentException`，而非深層的 IO 例外。
  2. 收斂前端 GitHub Token 依賴。移除 `GitHubConfig.tsx` 中的「Repository Access Key」輸入框，改為全面依賴後端的 `GITHUB_ORG_TOKEN`。
  3. 移除了前端多餘的 `token` API 參數，並從測試伺服器寫入有效的 GitHub Token。

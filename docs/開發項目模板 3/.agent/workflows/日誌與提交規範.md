---
description: 規範化開發任務完成後的日誌記錄與代碼提交流程
---

當完成一個開發任務或修復一個 Bug 時，請嚴格遵守以下流程：

1. **確認時間**
   - 使用 `date` 指令確認當前的系統時間。

2. **記錄詳細問題與解決方式 (僅針對 Bug 修復/問題排查)**
   - 如果任務涉及問題修復，必須在 `devlog/問題與解決-devlog.md` 中新增紀錄。
   - **格式要求**：
     ```markdown
     YYYY-MM-DD HH:MM — [標題]

     問題描述
     - 場景：...
     - 錯誤訊息：...

     原因分析
     - ...

     解決方案
     - ...

     驗證結果
     - ...

     紀錄時間：HH:MM
     ```
   - **操作規範**：必須使用 `replace_file_content` 或 `multi_replace_file_content` 工具在檔案適當位置插入，**禁止使用 `cat >>` 指令追加**。

3. **更新測試反饋狀態 (若適用)**
   - 若此任務解決了 `devlog/測試反饋.md` 中的問題，必須在該檔案對應的問題點標記「已解決」並記錄日期與時間。

4. **記錄今日日誌摘要**
   - 在 `devlog/YYYY-MM-DD.md` (根據當日日期) 中新增該任務的摘要。
   - **格式要求**：
     ```markdown
     ## [標題] [HH:MM]

     ### 完成功能 / 修復內容
     - [x] **[重點摘要]**：
         - ...

     ### 提交內容
     - ...
     ```
   - **操作規範**：使用編輯工具插入內容，確保 Markdown 結構正確。

5. **提交代碼 (Git Commit & Push)**
   - 執行 `git add .`
   - 執行 `git commit -m "[Type]: [Subject]..."`
   - 執行 `git push`
   - 獲取提交的 Short Hash (例如 `a1b2c3d`)。

6. **更新提交記錄**
   - 回到 `devlog/YYYY-MM-DD.md`，在剛剛新增的摘要區塊中，標註 Commit ID。
   - 範例：`### 提交內容 (Commit: a1b2c3d)`
   - 再次 `git add` 該日誌檔並使用 `git commit --amend` 更新日誌，最後 `git push -f` (若多人協作則使用新 commit)。

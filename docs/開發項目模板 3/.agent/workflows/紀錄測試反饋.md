---
description: 協助用戶將測試反饋整理並記錄到 devlog/測試反饋.md
---

1. **確認輸入資料**
   - 接收用戶提供的原始反饋文本。
   - 確認當前日期與時間。

2. **分析現有檔案**
   - 讀取 `devlog/測試反饋.md`。
   - 檢查目前已有的反饋次數（例如目前是 "第一次反饋"，下一次就是 "第二次反饋"）。

3. **格式化內容**
   - 標題層級：
     ```markdown
     
     ## 第 N 次反饋
     日期：YYYY-MM-DD
     時間：HH:mm
     ```
   - 項目層級（針對每個問題點，依照用戶提供的編號）：
     - 標題：`### X. [問題簡述]`
     - 屬性：
       - `- 平台：[Web App / App (Android/iOS)]` (若未提供則根據內容推測或標註 "待確認")
       - `- 端點：[管理端 / 員工端]` (若未提供則根據內容推測或標註 "待確認")
       - `- 內容：[詳細描述]`

4. **寫入檔案**
   - 將格式化後的內容**追加 (Append)** 到 `devlog/測試反饋.md` 的末尾。
   - 確保不同次反饋之間有空行分隔。

   - 完成後通知用戶已記錄完成，並提供檔案連結。

6. **標記問題已解決**
   - 當開發者修復了某個反饋問題時，應更新 `devlog/測試反饋.md`。
   - **格式**：在該問題的內容下方，追加一行引用文字：
     `> **[已解決]** 解決時間：YYYY-MM-DD HH:mm`
